# Heroes

## Описание проекта
Проект посвящён разработке ключевых алгоритмов для автобаттлера вдохновленного игрой Heroes of Might and Magic 3. <br>
Реализованы 4 основных алгоритма:

1. Генерация армии противника (`GeneratePreset`)
2. Симуляция пошагового боя (`SimulateBattle`)
3. Определение доступных целей для атаки (`SuitableForAttackUnitsFinder`)
4. Поиск кратчайшего пути между юнитами (`UnitTargetPathFinder`)

## Реализованные алгоритмы и анализ сложности

### 1. GeneratePreset - Генерация армии противника

Алгоритм: Жадный алгоритм с приоритетом по эффективности

Сложность:
- Сортировка: O(t log t), где t = количество типов юнитов (в нашем случае 4)
- Заполнение армии: O(m), где m = максимальное число юнитов в армии
- Итого: O(t log t + m)

### 2. SimulateBattle - Симуляция пошагового боя

Реализация:
1. Инициализация: Все юниты объединяются и сортируются по убыванию атаки
2. Основной цикл: Пока в обеих армиях есть живые юниты:
   - Каждый раунд проходит по всем юнитам (включая мертвых)
   - Мертвые юниты пропускаются проверкой `isAlive()`
   - Живые юниты атакуют через метод `attack()` своей программы


Сложность: <br>
Пусть: <br>
  n = общее количество юнитов <br>
  k = количество живых юнитов в раунде (k ≤ n) <br>
  R = количество раундов <br>

Операции:
1. Сортировка всех юнитов: O(n log n) - один раз
2. Каждый раунд: проход по всем n юнитам → O(n)
3. В каждом раунде для живых юнитов: attack() за O(1)

Общая сложность = O(n log n) + R × O(n)
                 ≤ O(n log n) + n × O(n)
                 = O(n²)


### 3. UnitTargetPathFinder - Поиск кратчайшего пути до цели

Алгоритм: BFS (поиск в ширину)

Сложность:
- Создание карты препятствий: O(n), n = количество юнитов
- BFS: O(V + E), где:
   - V = W × H = 27 × 21 = 567 вершин
   - E ≤ 8V = 4536 рёбер
